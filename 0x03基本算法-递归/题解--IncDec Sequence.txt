最少操作次数：

每次操作可以同时改变两个差分值（一个加1，一个减1）。

设 pos_sum 为 d[2..n] 中所有正数的和，neg_sum 为 d[2..n] 中所有负数的绝对值和。

我们可以先通过 min(pos_sum, neg_sum) 次操作将正负数配对消除。

剩余 |pos_sum - neg_sum| 个非零值需要单独处理，每次操作只能消除一个。

因此，最少操作次数为 max(pos_sum, neg_sum)


为什么结果是 |pos_sum - neg_sum| + 1？
情况1：pos_sum > neg_sum（剩余正数）
配对后剩余 k = pos_sum - neg_sum 个单位的正数需要处理。

处理每个单位正数有两种方式：

与 d[1] 配对：对区间 [1, i-1] 加1

效果：d[1] 增加1，该正数减少1

示例：从 [1, 1, 2, 2] 到 [2, 2, 2, 2]

单独处理：对区间 [i, n] 减1

效果：该正数减少1，d[1] 不变

示例：从 [1, 1, 2, 2] 到 [1, 1, 1, 1]

数学推导：

设我们选择 x 个单位与 d[1] 配对（0 ≤ x ≤ k）

那么 d[1] 最终值 = 初始值 + x

x 有 k+1 种选择（0, 1, 2, ..., k）

因此有 k+1 种不同的最终数列
